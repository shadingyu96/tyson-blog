Playwright 同步与异步机制深度解析

2026-03-06 · 进阶

1) Playwright 是什么

Playwright 是微软推出的浏览器自动化框架，支持 Chromium / Firefox / WebKit，常用于 E2E 测试、自动化操作、页面采集与截图。

2) 底层执行机制：本质上都是异步 I/O

无论你写的是“同步风格”还是“异步风格”，Playwright 底层都需要与浏览器进程异步通信。同步 API 只是把异步流程包装成阻塞式调用，提升可读性。

3) Node.js：天然异步（Promise + await）

const { chromium } = require('playwright');

(async () => {
  const browser = await chromium.launch();
  const page = await browser.newPage();
  await page.goto('https://example.com');
  console.log(await page.title());
  await browser.close();
})();

Node 版本几乎所有操作都返回 Promise，忘记 await 会造成竞态问题。

4) Python：同时提供 sync 与 async 两套 API

同步 API（易读）

from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page()
    page.goto("https://example.com")
    print(page.title())
    browser.close()

异步 API（并发友好）

import asyncio
from playwright.async_api import async_playwright

async def main():
    async with async_playwright() as p:
        browser = await p.chromium.launch()
        page = await browser.new_page()
        await page.goto("https://example.com")
        print(await page.title())
        await browser.close()

asyncio.run(main())

5) 自动等待机制（Auto-wait）

Playwright 会在点击、输入等操作前自动等待元素满足可见、可交互、稳定等条件，减少 flaky tests 和硬编码 sleep。

6) 什么时候用同步，什么时候用异步？

  • 线性脚本、快速开发：优先同步 API（Python）
  • 高并发任务、异步服务集成：优先异步 API
  • Node 环境：默认走 async/await 心智模型

7) 常见坑

  1. 忘记 await 导致执行顺序错乱
  2. 在已有事件循环里硬套同步 API
  3. 用 sleep 代替条件等待，导致慢且不稳定
  4. 并发开太多页面，资源占用飙升

8) 总结

Playwright 的底层是异步通信；同步只是对异步的友好封装。选型核心不是“哪个更高级”，而是“哪个更匹配你的任务复杂度和并发需求”。
