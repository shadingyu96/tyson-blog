很多人听到 AI 术语会头大。其实它们没那么玄，像搭积木一样就能理解。

## 先说结论（30秒版）

- **Agent（智能体）**：会“办事”的 AI 助手，不只是聊天。
- **MCP**：AI 连接外部工具的“统一插口标准”。
- **Skill（技能）**：某类任务的“现成方法包/说明书”。

一句话：你提需求 → Agent 想步骤 → Skill 给方法 → MCP 连工具 → Agent 给结果。

## Agent 是什么？

普通 AI 像“会回答问题的百科全书”，而 Agent 更像“会干活的助理”。

1. 听懂目标
2. 拆分步骤
3. 调用工具
4. 检查结果
5. 返回结论与建议

## MCP 是什么？

MCP（Model Context Protocol）可以理解成 AI 世界的“USB 接口标准”。

以前每个工具都要单独适配；有了 MCP，只要工具支持 MCP，Agent 就能用统一方式连接。

## Skill 是什么？

Skill 就是一份任务说明书，告诉 Agent：

- 什么时候用这套方法
- 先做什么、后做什么
- 该用哪些工具
- 要避开什么坑

## 三者关系（一句话）

**Agent 是执行者，Skill 是打法，MCP 是连接器。**

## 出错时怎么办？

1. 先重试（临时网络问题很常见）
2. 再换备用工具
3. 必要时降级输出（先给可用结果）
4. 最后向用户补充提问

## 记忆口诀

**Agent 是人，Skill 是方法，MCP 是插头。**
